# RAGAS Judge Calibration Analysis

The RAGAS Judge Calibration Analyzer is a diagnostic workflow tool designed to interpret local RAGAS evaluation results. When running local evaluation with smaller LLMs (like `qwen2.5-coder:3b` via Ollama), RAGAS metrics (especially `context_precision` and `faithfulness`) can experience parser instability or consistently report `0.0` scores even when the underlying retrieval system successfully finds the correct files.

To distinguish between **judge limitations** and **actual retrieval/context issues**, the calibration analyzer computes a set of deterministic, local retrieval metrics alongside RAGAS scores.

---

## Deterministic Context-File Diagnostics

These deterministic metrics are **not** generated by the LLM judge. Instead, they are calculated programmatically by comparing the trace's retrieved contexts with the golden queries' expected files (using normalized paths).

### Per-Query Fields

For each query row in `.per_query` / `.queries`, the following diagnostic fields are populated:

1. **`expected_context_file_hit`** (boolean)
   - `true` if at least one expected file appears in the retrieved files.
   - If a query lists no `expected_files`, it defaults to `true`.

2. **`expected_context_file_rank`** (int | null)
   - The 1-based index/rank of the first retrieved expected file.
   - If no expected files are found, or if `expected_files` is empty, it is `null`.

3. **`expected_context_file_precision`** (float | null)
   - Calculated as: `(Count of unique expected files retrieved) / (Total unique files retrieved)`.
   - If `expected_files` is empty, it is `null`.

4. **`expected_context_file_reciprocal_rank`** (float | null)
   - Calculated as: `1.0 / expected_context_file_rank`.
   - If the expected file is missing or `expected_files` is empty, it is `null`.

5. **`expected_context_files_found`** (list of strings)
   - The list of original expected file strings that were successfully retrieved.

6. **`expected_context_files_missing`** (list of strings)
   - The list of original expected file strings that were not retrieved.

---

## Path Normalization Rules

To match query expectations with database paths regardless of monorepo folder adjustments or environment paths, both expected and retrieved file paths are normalized before comparison:
- Backward slashes `\` are converted to forward slashes `/`.
- Leading `./` and trailing slashes are stripped.
- Leading `backend/` prefixes are stripped.

This treats `"backend/retrieval/config.py"` and `"retrieval/config.py"` as equivalent for matching, while preserving the original strings in reports.

---

## Interpretation and Calibration Rules

The analyzer flags discrepancies using automated interpretation statements appended to the `interpretation` field:

- **RAGAS Disagreement**: If `context_precision` is `0.0` but `expected_context_file_hit` is `true`, the analyzer adds:
  `"RAGAS context_precision disagrees with deterministic expected-file hit"` (which helps isolate LLM judge scoring failures from real retrieval bugs).
- **Rank 1 Success**: If the expected file is retrieved first, it adds:
  `"expected file ranked first"`.
- **Sub-optimal Rank**: If the expected file is retrieved but below rank 1, it adds:
  `"expected file retrieved below rank 1"`.
- **Retrieval Missing**: If expected files are defined but none are hit, it adds:
  `"expected file missing from retrieved contexts"`. This condition also triggers `retrieval_tuning_recommended = true`.

---

## Top-Level Aggregated Metrics

The JSON report outputs overall deterministic context-file statistics under the top-level `"deterministic_context_file_diagnostics"` key:

```json
"deterministic_context_file_diagnostics": {
  "queries_with_expected_files": 3,
  "expected_file_hit_count": 2,
  "expected_file_hit_rate": 0.6666666666666666,
  "mean_expected_file_rank": 2.0,
  "mean_expected_file_reciprocal_rank": 0.6666666666666666,
  "mean_expected_context_file_precision": 0.4444444444444444
}
```

- **`queries_with_expected_files`**: Total number of queries evaluated that specified at least one expected file.
- **`expected_file_hit_count`**: Number of those queries that successfully retrieved at least one expected file.
- **`expected_file_hit_rate`**: Hit rate fraction (`hit_count / queries_with_expected_files`).
- **`mean_expected_file_rank`**: The average rank of the first expected file across all successful hits.
- **`mean_expected_file_reciprocal_rank`**: The Mean Reciprocal Rank (MRR) across all successful hits.
- **`mean_expected_context_file_precision`**: The average deterministic precision across all queries with expected files.
